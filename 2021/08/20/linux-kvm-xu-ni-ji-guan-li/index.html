一、禁用Linux服务器密码登录

  • 1.本地ssh目录(Windows为C:\Users\lzww.ssh；linux为/root/.ssh)的公钥复制放到服务器/root/.ssh/authorized_keys文件中，若没有该文件则新建；
  • 2.修改/etc/ssh/sshd_config配置文件，设置PubkeyAuthentication为yes,设置AuthorizedKeysFile为上一步的路径
  • 3.设置PasswordAuthentication为no，禁止密码方式登录，然后重启ssh服务(systemctl restart sshd)即可。
  • 4.如果是非root用户，配置完以上流程后，还需要修改文件权限：
    • 先调整文件夹~/.ssh的权限：chmod 700 ~/.ssh
    • 再调整文件~/.ssh/authorized_keys的权限：chmod 600 ~/.ssh/authorized_keys

二、通过LVM挂载磁盘

1.扫描硬盘分区

lvmdiskscan;
扫描结果中带有 LVM physical volume 的就是物理分区，其对应的分区名可以直接创建pv
如果磁盘未分区，则：
 a.查看磁盘分区情况：fdisk -l或者lsblk
 b.新建分区：
	 fdisk /dev/sda
  	 m 查看帮助  
  	 p 打印现有分区  
  	 n 新建分区  
	   w 保存
	   mkfs -t ext4 /dev/sda1 格式化新建的分区，分区需要格式化以后才能挂载
 c.删除分区(若已挂载需要先卸载)
	 fdisk /dev/sdc
	   m
	   d 删除分区
	   1 选择分区编号
	   d 确认删除
	   w 保存

2.创建物理卷PV

pvcreate [device1]；
pvcreate /dev/sda{1,2,3,4} #给sda1-4 4个分区创建pv
例如：pvcreate /dev/sda1
查看pv是否存在：pvscan
查看物理卷信息：pvdisplay
删除pv:pvremove /dev/sda1

3.创建卷组VG

vgcreate VG_NAME [device1] [device2] ...
例如：vgcreate vgroup /dev/sda1
也可以先创建卷组，然后往卷组里面添加PV；
添加pv：vgextend VolGroup /dev/sda1
查看卷组信息：vgdisplay
删除卷组：vgremove VG_NAME

4.创建逻辑卷LV

lvcreate -L 200G -n lv_data vgroup
含义：在卷组vgroup中创建一个大小200G的逻辑卷lv_data
查看逻辑卷信息：lvdisplay
删除逻辑卷：lvremove /dev/vgroup/lv_data(使用lv path进行删除)
注意：需要先卸载掉挂载的卷才能删除逻辑卷

5.格式化逻辑卷LV

mkfs.ext4 /dev/vgroup/lv_data
注意：格式化LV是使用的是LV PATH，而不是NAME;尽量不要使用ext*方式去格式化，而是使用xfs
mkfs.xfs -f /dev/sdb1

6.挂载逻辑卷

mount /dev/vgroup/lv_data /data
含义：挂载逻辑卷/dev/vgroup/lv_data到/data目录
umount /dev/vgroup/lv_data
含义：卸载已挂载的逻辑卷(卸载以后该逻辑卷可以重新挂载)

7.扩展逻辑卷大小

a.卸载掉挂载的卷：umount /data
b.重新设置逻辑卷的大小为300G：lvresize -L 300G /dev/vgroup/lv_data
c.检查磁盘错误： e2fsck -f /dev/vgroup/lv_data
d.扩展文件系统,更新ext4信息：resize2fs /dev/vgroup/lv_data
e.再次挂载逻辑卷：mount /dev/vgroup/lv_data /data

8.开机自动挂载

编辑/etc/fstab：vi /etc/fstab
添加信息：
/dev/VolGroup/lv_data   /data     ext4    defaults        0 0
/dev/vgroup/lv_data     /data     xfs    defaults        0 0
# 参数说明：
<file system>	<dir>	<type>	<options>	<dump>	<pass>
<file systems> - 要挂载的分区或存储设备.
<dir> - <file systems>的挂载位置。
<type> - 要挂载设备或是分区的文件系统类型，支持许多种不同的文件系统：ext2, ext3, ext4, reiserfs, xfs, jfs, smbfs, iso9660, vfat, ntfs, swap 及 auto。 设置成auto类型，mount 命令会猜测使用的文件系统类型，对 CDROM 和 DVD 等移动设备是非常有用的。
# centos7.0开始默认文件系统是xfs，centos6是ext4，centos5是ext3(尽量使用xfs)
<options> - 挂载时使用的参数，注意有些 参数是特定文件系统才有的。一些比较常用的参数有 (mount(8))：
auto - 在启动时或键入了 mount -a 命令时自动挂载。
noauto - 只在你的命令下被挂载。
exec - 允许执行此分区的二进制文件。
noexec - 不允许执行此文件系统上的二进制文件。
ro - 以只读模式挂载文件系统。
rw - 以读写模式挂载文件系统。
user - 允许任意用户挂载此文件系统，若无显示定义，隐含启用 noexec, nosuid, nodev 参数。
users - 允许所有 users 组中的用户挂载文件系统.
nouser - 只能被 root 挂载。
owner - 允许设备所有者挂载.
sync - I/O 同步进行。
async - I/O 异步进行。
dev - 解析文件系统上的块特殊设备。
nodev - 不解析文件系统上的块特殊设备。
suid - 允许 suid 操作和设定 sgid 位。这一参数通常用于一些特殊任务，使一般用户运行程序时临时提升权限。
nosuid - 禁止 suid 操作和设定 sgid 位。
noatime - 不更新文件系统上 inode 访问记录，可以提升性能(参见 atime 参数)。
nodiratime - 不更新文件系统上的目录 inode 访问记录，可以提升性能(参见 atime 参数)。
relatime - 实时更新 inode access 记录。只有在记录中的访问时间早于当前访问才会被更新。（与 noatime 相似，但不会打断如 mutt 或其它程序探测文件在上次访问后是否被修改的进程。），可以提升性能(参见 atime 参数)。
flush - vfat 的选项，更频繁的刷新数据，复制对话框或进度条在全部数据都写入后才消失。
defaults - 使用文件系统的默认挂载参数，例如 ext4 的默认参数为:rw, suid, dev, exec, auto, nouser, async.
<dump> dump 工具通过它决定何时作备份. dump 会检查其内容，并用数字来决定是否对这个文件系统进行备份。 允许的数字是0和1。0表示忽略，1则进行备份。大部分的用户是没有安装 dump 的 ，对他们而言 <dump> 应设为 0。
<pass> fsck 读取 <pass> 的数值来决定需要检查的文件系统的检查顺序。允许的数字是0,1,和2。根目录应当获得最高的优先权1,其它所有需要被检查的设备设置为2。0表示设备不会被fsck所检查。

9.LVM概念

a.PE
Linux文件系统中数据存储的最小单位是block，而PE的作用就像block一样，是Lvm数据存储的最小单位，默认是4MB ，一个卷组VG最多能有65534个PE。

b.物理卷PV
物理卷PV可以是硬盘分区，也可以是硬盘本身或回环文件，是Lvm的基本存储逻辑块。物理卷包括一个特殊的header，其余部分被分割为一块块物理区域。

c.卷组VG
将一个物理卷或多个物理卷组合成为一个管理单元，该管理单元就是卷组。卷组就相当于正常的文件系统中的硬盘，可在上面创建一个或多个Lvm分区。

d.逻辑卷LV
逻辑卷就像正常的硬盘分区，用户可以在逻辑卷上格式化创建文件系统。

10.Linux查看磁盘信息

df -h
相关命令：
df -hl：查看磁盘剩余空间
du -sh [目录名]：返回该目录的大小(在文件夹中可以不用指定目录名)
du -sm [文件夹]：返回该文件夹总M数
du -h  [目录名]：查看指定文件夹下的所有文件大小（包含子文件夹）

11.总结

LVM中:
物理分区<->PV #一个磁盘分区对应一个PV
PV1,PV2,...,PVn <-> VG #多个PV组合成一个VG
VG <-> LV1,LV2,...,LVn #1个VG分成多个LV
LV <-> /home  #LV格式化以后就可以挂载到用户目录去使用了
  
不使用LVM进行分区管理时：分区-->文件系统，使用LVM后变成了：分区-->PV--VG---LV--文件系统；在此过程中把分区划分成了很多个磁盘来使用，这样管理起来更加灵活，而且对LV进行扩容很方便，还不影响文件里面的数据。

三、CentOS配置KVM-server

1.检查cpu是否支持虚拟化

$ grep vmx /proc/cpuinfo
如果有很多输出信息说明支持

2.开启虚拟化

$ lsmod | grep kvm 检测是否开启虚拟化

3.安装管理网桥的工具

$ yum -y install bridge-utils
$ systemctl restart network

4.安装libvirt + kvm

# libvirt是管理虚拟机的API库
$ yum -y install libcanberra-gtk2 qemu-kvm.x86_64 qemu-kvm-tools.x86_64  libvirt.x86_64 libvirt-cim.x86_64 libvirt-client.x86_64 libvirt-java.noarch  libvirt-python.x86_64 libiscsi-1.7.0-5.el6.x86_64  dbus-devel  virt-clone tunctl virt-manager libvirt libvirt-python python-virtinst
最少安装：yum -y install qemu-kvm libvirt virt-install bridge-utils

5.开启kvm服务

$ systemctl enable libvirtd
$ systemctl start libvirtd

6.桥接网卡

# 桥接物理机网卡和虚拟机网卡
a.复制一份物理机网卡信息，重命名为br0：
$ cd /etc/sysconfig/network-scripts
$ cp ifcfg-p8p1 ifcfg-br0
b.编辑桥接网卡信息：
    TYPE=Bridge
    BOOTPROTO=static
    NAME=br0
    DEVICE=br0
    ONBOOT=no
    IPADDR=172.18.22.240
    GATEWAY=172.18.22.1
    NETMASK=255.255.255.0
桥接网卡里面IP配的是物理机的IP，然后在虚拟机里面再通过桥接到br0的方式来连接到物理机
c.在物理机网卡里面添加桥接网卡：(如果配置桥接则物理机网卡里面IP相关的信息不用配，不会生效)
    TYPE=Ethernet
    BOOTPROTO=static
    NAME=p8p1
    DEVICE=p8p1
    ONBOOT=yes
    IPADDR=172.18.22.240
    GATEWAY=172.18.22.1
    NETMASK=255.255.255.0
    BRIDGE=br0
d.网卡配置完成后需要重启网络服务：
$ systemctl restart network
(网卡DNS解析:vi /etc/resolv.conf；配置114.114.114.114)

7.下载系统镜像

下载好系统镜像文件，放在目录/data/kvm/vmimages下；同时创建虚拟硬盘的路径/data/kvm/{vm-name}

8.创建虚拟机

$ virt-install 
--name=lzw-243 
--memory=2048,maxmemory=4096 
--vcpus=2,maxvcpus=4 
--os-type=linux 
--os-variant=rhel7 
--location=/data/kvm/vmimages/CentOS-7-x86_64-Minimal-1810.iso 
--disk path=/data/kvm/lzw-243/lzw-243.img,size=100 
--bridge=br0 
--graphics=none 
--console=pty,target_type=serial  
--extra-args="console=tty0 console=ttyS0"
# 建议虚拟机名字后面加上 IP 地址末位用以标识。如：lzw-243
命令说明：
# name 指定虚拟机的名称
# memory 指定分配给虚拟机的内存资源大小
# maxmemory 指定可调节的最大内存资源大小，因为KVM支持热调整虚拟机的资源
# vcpus 指定分配给虚拟机的CPU核心数量
# maxvcpus 指定可调节的最大CPU核心数量
# os-type 指定虚拟机安装的操作系统类型
# os-variant 指定系统的发行版本
# location 指定ISO镜像文件所在的路径，支持使用网络资源路径，也就是说可以使用URL
# disk path 指定虚拟硬盘所存放的路径及名称，size则是指定该硬盘的可用大小，单位是G
# bridge 指定使用哪一个桥接网卡，也就是说使用桥接的网络模式
# graphics 指定是否开启图形
# console 定义终端的属性，target_type则是定义终端的类型
# extra-args 定义终端额外的参数
  
$ 安装win7虚拟机
# 先创建虚拟硬盘文件
& qemu-img create -f img win7.img 100G
# 创建虚拟机
$ virt-install --virt-type kvm --name win7 --ram 2048 --cdrom=/data/kvm/vmimages/cn_win7_x86.iso  --boot cdrom --cpu core2duo --network bridge=br0,model='virtio' --graphics vnc,password=passw0rd,listen=0.0.0.0,port=5900 --disk path=/data/kvm/win7/win7.img,bus='ide',size=100 --noautoconsole --os-type=windows
#执行安装完需要通过vnc来进行图形界面安装，安装以后要配置ip地址及DNS，直接指定即可
#注意：需要先安装图形化工具 yum install -y virt-viewer

9.系统安装

# 执行完虚拟机创建以后，会进入到系统安装界面，在此过程中，带[!]的的选项都是需要进行配置的，通过输入选项左边的数字进行相应的配置，一般需要配置语言、时区、安装源、网络、密码等信息。配置完以后系统就开始安装，安装完回车会重启进入虚拟机登录界面，登录虚拟机以后，进行IP配置。

10.配置虚拟机静态IP

# 查看虚拟机网卡，一般是eth0
$ ip addr
# 进入网卡配置目录，修改配置信息
$ cd /etc/sysconfig/network-scripts
$ vi ifcfg-eth0
# 在里面添加以下信息
a.BOOTPROTO项设置为static
b.ONBOOT项修改为yes
c.添加以下3项信息：
	IPADDR=172.18.22.243
    GATEWAY=172.18.22.1
    NETMASK=255.255.255.0
d.修改完以后重启网络服务
$ systemctl restart network
# 然后通过$ ip addr查看到IP信息说明网卡配置成功，虚拟机创建完成

# 注意：如果ping不通外网则需要设置DNS服务器
$ vi /etc/resolv.conf
# 添加以下信息
  nameserver 114.114.114.114
  nameserver 8.8.8.8

11.虚拟机管理(物理机中)

# 查看物理机上所有虚拟机
  $ virsh list --all
# 查看虚拟机磁盘文件所在目录
  $ virsh domblklist vm-name
# 清除虚拟机
  $ virsh destroy vm-name   #停止虚拟机
  $ virsh undefine vm-name
  $ rm -rf <directory-of-vm-name>(/data/kvm/lzw-243;虚拟硬盘存放的路径)
# 查看虚拟机配置文件
  $ ls /etc/libvirt/qemu/
  $ ls /etc/libvirt/qemu/networks/
  $ ls /etc/libvirt/qemu/networks/autostart/
# 进入指定的虚拟机，进入的时候还需要按一下回车
  $ virsh console lzw-243 
# 启动虚拟机
  $ virsh start lzw-243   
# 关闭虚拟机
  $ virsh shutdown lzw-243   
# 强制停止虚拟机
  $ virsh destroy lzw-243   
# 彻底销毁虚拟机，会删除虚拟机配置文件，但不会删除虚拟磁盘
  $ virsh undefine lzw-243   
# 设置宿主机开机时该虚拟机也开机
  $ virsh autostart lzw-243   
# 解除开机启动
  $ virsh autostart --disable lzw-243

12. VNC登录kvm虚拟机

当虚拟机访问异常，无法通过ssh协议正常登录甚至不能ping通的时候，便需要借助VNC远程控制台的方式进行登录查看。登录需要进行的操作如下：

# 1，宿主机放开VNC监听
vi /etc/libvirt/qemu.conf
vnc_listen = "0.0.0.0"
systemctl restart libvirtd
 
 
# 2, 宿主机暂时关闭防火墙和SELinux服务,如果已经满足，可以跳过
systemctl stop firewalld
setenforce 0
 
 
# 3，关闭要登录的虚拟，并且编辑模板文件
virsh destroy [VM-ID]
virsh edit [VM-ID] # 如果不介意使用自动生成的端口，这一步可以跳过，port根据情况自定义即可
<graphics type='vnc' port='5910' autoport='no' listen='0.0.0.0'>     
  <listen type='address' address='0.0.0.0'/>      
</graphics>
 
 
# 4，启动虚机
virsh start [VM-ID]

13. 修改主机名

# 查看主机名相关的设置
  $ hostnamectl
# 修改主机名
  $ hostnamectl set-hostname lzw-243
# 修改好需要更新配置文件，否则机器重启会很慢
  $ vi /etc/hosts      
   127.0.0.1  lzw-243
   ::1        lzw-243
 （注释掉原来的配置，更新为新的主机名即可）
